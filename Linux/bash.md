## 🔥 Основы синтаксиса

### Переменные
```bash
# Присвоение (без пробелов вокруг =)
var="value"
num=42

# Использование
echo "$var"
echo "${var}"       # Явное отделение от текста: ${var}123

# Вывод команды в переменную
now=$(date)
files_count=$(ls | wc -l)

# Арифметика (только целые числа)
result=$((num + 10))
```

### Условия и проверки
```bash
if [[ "$var" == "yes" ]]; then
  echo "Yes!"
elif [[ $num -gt 10 ]]; then
  echo ">10"
else
  echo "Other"
fi

# Частые условия:
# -z "$str"           пустая строка
# -n "$str"           не пустая
# -f "file"           обычный файл существует
# -d "dir"            директория существует
# -x "file"           исполняемый
# -eq, -ne, -lt, -gt  сравнение чисел
# ==, !=              сравнение строк
# =~                  regex: [[ "$text" =~ ^start ]]
```

### Циклы
```bash
# По списку слов
for i in one two three; do
  echo "$i"
done

# По диапазону чисел
for i in {1..5}; do
  echo "$i"
done

# По файлам (без проблем с пробелами)
while IFS= read -r line; do
  echo "$line"
done < file.txt

# Бесконечный цикл с прерыванием
while true; do
  sleep 1
  [[ -f "/tmp/stop" ]] && break
done
```

### Функции
```bash
my_func() {
  local param1=$1      # первый аргумент
  local result="Hello, $param1"
  echo "$result"       # возврат через echo (захватывается $())
}

greeting=$(my_func "Bro")
echo "$greeting"
```

### Подстановки и дефолтные значения
```bash
echo "${var:-default}"   # если var не задана, подставить default (сама var не меняется)
echo "${var:=default}"   # если не задана — присвоить default
echo "${var:?message}"   # если не задана — ошибка и выход
echo "${var:+set}"       # если задана — подставить set, иначе пусто

# Вырезка строки
name="file.txt"
base="${name%.*}"        # file (убирает суффикс .*)
ext="${name##*.}"        # txt  (убирает префикс *.)
```

## 💎 Самые нужные команды

### Работа с файлами и путями
- `basename /path/to/file.txt` → `file.txt`
- `dirname /path/to/file.txt` → `/path/to`
- `readlink -f file` — полный путь
- `mktemp` — временный файл/директория
- `test -f file && echo "exists"`

### Текст и фильтры
- `grep`, `egrep` (или `grep -E`) — поиск строк  
- `sed` — замена, удаление  
  `sed 's/old/new/g' file`  
  `sed -i 's/old/new/g' file` — на месте
- `awk` — обработка столбцов  
  `awk '{print $1, $3}' file`  
  `awk -F: '{print $1}' /etc/passwd`
- `cut -d':' -f1` — вырезать поле
- `sort`, `uniq`, `wc -l`, `head`, `tail`
- `tr` — замена символов (верхний/нижний регистр)
- `jq` — обработка JSON (надо ставить отдельно)

### Сеть и удалённые вызовы
- `curl -sSfL http://...` (тихий, ошибки, follow redirects)
- `wget -qO- http://...` (скачать в stdout)
- `nc -zv host port` — проверка порта
- `ping -c 1 -W 2 host` — один пинг с таймаутом

### Процессы и управление
- `ps aux | grep name`
- `pgrep -f "pattern"` — PID по имени
- `kill`, `pkill`
- `timeout 5 command` — запуск с таймаутом
- `command &` — запустить в фоне
- `wait` — ждать завершения фоновых процессов
- `trap 'echo Interrupted; exit' INT` — перехватить Ctrl+C

### Информация о системе
- `date +%Y-%m-%d_%H-%M-%S` — метка времени
- `uname -s` (Linux) и т.п.
- `hostname -f`
- `df -h` / `du -sh dir` — место на диске
- `free -m` — память

## 🚦 Спецпеременные и коды выхода

- `$0` — имя скрипта
- `$1`, `$2`... — аргументы скрипта
- `$#` — количество аргументов
- `$@` — все аргументы по отдельности (в кавычках `"$@"`)
- `$*` — все аргументы одной строкой
- `$?` — код возврата последней команды (0 = успех)
- `$$` — PID текущего скрипта
- `$!` — PID последнего фонового процесса

Проверка успеха:
```bash
if command; then
  echo "ok"
fi
# или
command || { echo "fail"; exit 1; }
```

## 🔄 Перенаправления (помнить как таблицу умножения)

- `command > file` — перезаписать stdout в файл
- `command >> file` — добавить stdout в файл
- `command 2> file` — stderr в файл
- `command &> file` — и stdout, и stderr в файл (bash 4+)
- `command < file` — stdin из файла
- `command <<< "string"` — here-string (строка как stdin)
- `command << 'EOF' ... EOF` — heredoc
- `/dev/null` — чёрная дыра, чтобы скрыть вывод

## 🧩 Комбинирование команд

```bash
cmd1 && cmd2   # выполнить cmd2 только если cmd1 успешен
cmd1 || cmd2   # выполнить cmd2 только если cmd1 провалился
cmd1; cmd2     # последовательно в любом случае
cmd1 | cmd2    # pipe: stdout cmd1 -> stdin cmd2
```

## 📦 Полезные однострочники

```bash
# Выполнить команду для всех файлов по маске
for f in *.txt; do echo "Processing $f"; done

# Переименовать массово (например, .txt -> .bak)
for f in *.txt; do mv "$f" "${f%.txt}.bak"; done

# Скачать и распаковать
curl -sSfL https://example.com/file.tar.gz | tar xz

# Найти и заменить во всех файлах
find . -name "*.cfg" -exec sed -i 's/old/new/g' {} +

# Проверить, запущен ли скрипт от root
[[ $EUID -eq 0 ]] || { echo "Need root"; exit 1; }

# Логгирование с датой
log() { echo "$(date '+%F %T') $*"; }
log "Starting..."
```
