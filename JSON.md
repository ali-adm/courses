# JSON 
**(JavaScript Object Notation)** — это легкий формат обмена данными, который **легко читается и пишется людьми**, а также **легко парсится и генерируется машинами**. Он основан на подмножестве языков программирования JavaScript, но является независимым от языка, то есть **может использоваться в различных языках программирования**.

### [Back to the Future!](#back)


<a id="j_param"></a>
## Основные характеристики JSON:
- **Легковесность:** JSON использует простую и минималистичную структуру, что делает его компактным и легко читаемым.
- **Читаемость:** Благодаря синтаксису, напоминающему JavaScript, JSON легко понимаем и записываем человеком.
- **Совместимость:** JSON широко поддерживается различными языками программирования и платформами.

### [Back to the Future!](#back)


<a id="j_struct"></a>
## Структура JSON:
### JSON состоит из двух основных структур: объектов и массивов.

<a id="j_obj"></a>
### 1. Объекты:
Это **неупорядоченные** наборы пар `"ключ-значение"`. Объекты начинаются с `{` и заканчиваются `}`, а пары ключ-значение разделяются запятыми.
```json
{
  "ключ1": "значение1",
  "ключ2": "значение2"
}
```

- **Ключ:** Это строка, заключенная в двойные кавычки.
- **Значение:** Это может быть строка, число, объект, массив, true, false или null.

<a id="j_arr"></a>
### 2. Массивы:   
Это **упорядоченные** списки. Массивы начинаются с `[` и заканчиваются `]`, а элементы массива также разделяются запятыми.
```json
[
  "значение1",
  "значение2",
  "значение3"
]
```

<a id="j_prim"></a>
### 3. Примитивные значения:

**Строки**: Всегда заключены в двойные кавычки. Пример: "это строка".
**Числа**: Представляют собой целые или десятичные значения. Пример: 123, 12.34.
**Булевые значения**: true или false.
**Null**: Представляет отсутствие значения: null.

<a id="j_example"></a>
### Пример комплексного JSON:
```json
{
  "name": "Иван",
  "age": 30,
  "isStudent": false,
  "courses": ["математика", "информатика"],
  "address": {
    "street": "Пушкинская",
    "number": 12
  },
  "grades": null
}
```

- **Объект "address**": Вложенный объект с улицей и номером дома.
- **Массив "courses**": Перечисление изучаемых курсов.
- **Значение null для "grades"**: Указывает, что у объекта отсутствуют данные о оценках.

# XML
Давайте начнем с объяснения XML, а затем обсудим **различия** между **XML** и **JSON** и где каждый из них лучше использовать.

<a id="x_about"></a>
## Что такое XML?
**XML (Extensible Markup Language)** — это язык разметки, разработанный для **хранения и передачи данных** в формате, который легко **читается как людьми, так и машинами**. XML обеспечивает структуру данных с помощью тегов и атрибутов и позволяет пользователям создавать свои собственные теги, адаптивные к разным сценариям.

<a id="x_struc"></a>
### Структура XML:
Документ XML начинается с объявления (необязательно):
```xml
<?xml version="1.0" encoding="UTF-8"?>
```

<a id="x_el"></a>
### Элементы: 
Основные строительные блоки XML, которые начинают с начального и конечного тегов.
```xml
<note>
  <to>Татьяна</to>
  <from>Иван</from>
  <heading>Напоминание</heading>
  <body>Увидимся на встрече.</body>
</note>
```

<a id="x_attrib"></a>
### Атрибуты:
Это дополнительные свойства элементов и часто используются для передачи метаданных.
```xml
<note priority="high">
  <to>Татьяна</to>
  ...
</note>
```

<a id="x_hierarch"></a>
### Иерархия:
XML поддерживает вложенные элементы, создавая иерархическую структуру, которая отражает отношения между элементами.

<a id="j_vs"></a>
# Различия между JSON и XML:

## Синтаксис и читаемость:
- **JSON**: Имеет более простой и легковесный синтаксис. Легче читается и записывается человеком.
- **XML**: Более детализированный и гибкий в разметке, но может стать громоздким.

<a id="j_vs_format"></a>
### Форматирование данных:
- **JSON**: Использует ключи и значения для определения структуры данных.
- **XML**: Использует элементы (теги) и атрибуты.

<a id="j_vs_addon"></a>
### Возможности расширения:
- **JSON**: Имеет ограниченные возможности для расширения из-за простоты; его можно ### трактовать как просто структуру данных.
- **XML**: Очень гибок и поддерживает сложные схемы, включая пространственные имена.

<a id="j_vs_supp"></a>
### Поддержка типов данных:
- **JSON**: Имеет встроенные примитивные типы (строки, числа, логические значения, массивы и объекты).
- **XML**: Текстово-ориентирован и требует дополнительной обработки для интерпретации чисел и логических значений.

<a id="j_vs_use"></a>
### Использование пространств имен:
- **XML**: Поддерживает пространства имен, полезные для избегания конфликта имен.
- **JSON**: Не поддерживает пространства имен.

<a id="j_vs_where"></a>
# Где и что лучше использовать:

### JSON:
Идеально подходит для передачи данных между браузером и сервером благодаря своей **легковесности**.
Обширно используется в **веб-сервисах** и **API** из-за простоты и скорости обработки.
Подходит для **мобильных** приложений и **легковесных** систем.

### XML:
Предпочтителен в системах, где требуется **сложная документация данных** или **совместимость с существующими стандартами**.
Хорош для обмена данными между различными системами с **разнородными технологиями**.
Используется в **конфигурационных файлах** и если требуется **валидация данных** с помощью схем (XML Schema).
